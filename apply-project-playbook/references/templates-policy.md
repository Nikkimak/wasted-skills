# Templates Policy

Templates are accelerators, not authoritative sources.

## Use Templates For

- repeated canonical files with stable structure
- bootstrap docs that mostly vary by project details
- files where consistency matters more than originality

## Do Not Use Templates As

- a substitute for reading the canonical playbook
- a reason to create unnecessary docs
- a way to force `full` mode onto a small project

## Template Priority

Use these first:

- `project-agents-template.md`
- `context-readme-template.md`
- `project-state-template.md`
- `runtime-map-template.md`
- `repo-map-template.md`
- `decisions-readme-template.md`
- `deploy-runbook-template.md`
- `src-readme-template.md`

Use only when needed:

- `future-work-status-template.md`

## Adaptation Rule

Adapt templates to the actual project.
Do not leave placeholder structure that the project will not maintain.
